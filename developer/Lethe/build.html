Building the Project: A Developer’s Guide

This document is intended to guide developers on how to build existing programs and extend the build environment when needed.

Section 1: Building Existing Programs

1.1 Prerequisites

  • Setting Up Your Environment: Ensure that make is available in your system’s PATH. Make sure necessary environment variables such as DEVELOPER_HOME and EXECUTOR_IN_DIR are set. Source the env_build script to configure the environment: 
    source /path/to/env_build

1.2 Basic Build Process

  • How to Use Make: Developers can use the make command to build specific programs. For example: 
    make X
    Here, X is the name of the program you wish to build. If you do not specify any program, make will build the entire project: 
    make
  • Sample Commands: 
    make Arithmetic_Echo
    make Arithmetic_Syntax

1.3 Understanding the Output

  • After a successful build, executables will be created in the executor directory.
    The makefile-top.mk file will be the top-level control file for managing the build process.

Section 2: Advanced Usage for Developers

2.1 Adding New Programs

  • Steps to Add a New Program:
    • Modify makefile-project.mk and makefile-tool.mk to include the new program.
    • Add the program name to the EXECUTOR_IN_FL list in env_build.
    • Ensure that source files (e.g., .java or ANTLR grammar files) are properly placed in the relevant directories.

2.2 Advanced Make Commands

  • Using project-X and tool-Y: Developers can use advanced make commands to build only specific components. For example: 
    make project-Arithmetic_Syntax
    This command builds the Arithmetic_Syntax component only. Similarly, use make tool-X to build a specific tool: 
    make tool-Y

2.3 Debugging and Extending the Build Environment

  • Environment Variables: Developers can modify environment variables in the env_build script to add or remove programs, directories, or paths.
    Common variables include:
    • EXECUTOR_IN_FL: The list of programs to build.
    • ANTLR_OUT_DIR: Directory for ANTLR-generated files.
    • JAVA_COMP_IN_FPL: File list for Java compilation.
